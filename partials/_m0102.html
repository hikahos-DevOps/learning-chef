Goodbye, Traditional Scripting
CM is different from scripting.
• A CM tool should analyze the state of a server before making changes.
• For example, if the script dictates that Apache should be installed, but it already is, this step is skipped.
The Power of Idempotence
Why is idempotence important?
• If an unauthorized or accidental change is made to a server, idempotence will bring it back into alignment.
• The policy can be applied at regular intervals with no effect, unless a change is needed to bring the server back into alignment.

CM and the Cloud
What is the cloud?
• Compute resources that are owned by another entity
• These resources can be rented at a low cost and without commitment
• This provides nearly unlimited resources for anyone needing on-demand compute resources
Why use configuration management in the cloud?
• Provides for large-scale automation of compute resources
• Can use the public cloud's API, allowing for additional programmatic and graphical interfaces to be created in the cloud ecosystem
• Abstract cloud complexity with features provided by configuration management
